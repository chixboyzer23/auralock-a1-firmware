# Contributing to AURALOCK A1 Firmware

Thank you for your interest in contributing to the AURALOCK A1 firmware project.

We welcome developers and hardware enthusiasts who would like to improve the firmware or contribute new features.

---

## Reporting Issues

If you discover a bug or unexpected behavior, please open an issue on GitHub and include the following details:

Firmware version
ESP32 board model
RFID module used
Steps to reproduce the issue
Expected behavior
Actual behavior

Providing detailed information helps maintainers resolve issues faster.

---

## Suggesting Improvements

Developers are welcome to suggest improvements to the firmware including:

Performance optimizations
New hardware support
Bug fixes
Documentation improvements

Please open a GitHub issue describing your suggestion before submitting large changes.

---

## Submitting Pull Requests

To contribute code:

1. Fork the repository
2. Create a new branch for your changes
3. Implement and test your modifications
4. Submit a pull request with a clear description

Example branch naming:

```
feature/rfid-performance-improvement
```

---

## Coding Guidelines

When contributing code please follow these guidelines:

Use clear and readable C++ code
Follow standard Arduino coding practices
Add comments where necessary
Ensure existing firmware functionality is not broken
